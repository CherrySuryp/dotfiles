# Hello Command Example

Please greet me in a friendly way and provide a brief overview of what you can help with today.

## Best Practices Demonstrated:

1. **Description**: Commands should include a description in HTML comments at the top
2. **Tags**: Optional tags help organize and categorize commands
3. **Clear Instructions**: The command contains clear, specific instructions
4. **Structure**: Using markdown for readability (though plain text works too)
5. **Scope**: Keep commands focused on a single, well-defined task

## Tips for Creating Commands:

- Keep prompts concise and specific
- Use commands for frequently repeated tasks
- Commands can reference files, ask questions, or trigger workflows
- Access via `/hello` in the CLI
- Commands are just markdown files - the content is the prompt sent to Claude
